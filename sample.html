Operating System

An operating system is a system software that acts as an interface between the user and hardwares of a computer and reduces the internal complexity.

The main functionalitites of an operating system are as follows:

  • Process Management
  • Memory Management
  • File Management
  • Input/Output Management
  • Security Management

Program v/s Process

A program is a set of instruction that is given to the computer to execute a specific task.

A program under execution is called a process.

Stored Program Architecture

A stored program architecture is a type of computer design where programs are initially stored on a hard disk, which offers substantial storage capacity but slower access speeds. During execution, these programs are transferred to RAM, which, although having less storage capacity, provides much faster access speeds.

The schedulers involed in this are the following:

  • 1. Long-term schedulers- They determine which processes, from a larger pool, will be loaded into RAM for execution by the CPU.
  • 2. Short-term schedulers-They determine which one process, out of the multiple processes in RAM, will be executed first by the CPU.

The various types of Operating System

  1. 1. Serial Processing OS- In this type of OS the jobs are executed in a sequential manner in the order of First Come First Serve. In this the jobs are submited in the form of punch cards to a program counter which executes all instructions.
  2. 2. Batch Processing OS- In this similar types of jobs are grouped together and executed in a batch manner. There is a concept of resident monitor which manages the automatic sequencing of jobs.
  3. 3. Multi-Programming OS- In this type of OS we can execute multiple programs on the system at the same time. In this the CPU is never idle and it ensures maximum CPU utlization and maximum throughput.
    In this OS if one process other goes I/O operation the other process undergoes execution by the CPU.
  4. 4. Time Sharing OS- In time-sharing OS a process is alloted a certain period of time before which it has to complete its execution and after which it can be executed again. This short period of time is called Quantum Time.
  5. 5. Network OS- This types of OS connects multiple devices or computers on the network and allows them to share resources.
  6. 6. Distributed OS- In this the OS is separated into various sections which are loaded into multiple machines rather than a single machine.
  7. 7. Multi-Processor OS- In this type of OS the computer has multiple processors which can execute multiple programs at the same time.
  8. 8. Real Time OS- In this OS the process have to be executed before a certain deadline. It is of the following types,
    • 1. Hard RTOS- In this if the process completes after a given deadline the entire system crash. They are used in aircrafts and missile systems.
    • 2. Firm RTOS- In this if the process completes after a given deadline the output may not be useful but the result may be useful. For Example- Video Conference
    • 3. Soft RTOS- In this if the process completes after a given deadline the importance of the output decreses. For Example- Internet browsing.

Different architectures of Operating Systems

  1. 1. Layered Structure-In this the operating system is divided into several modules or layer. The beauty of this syetem is that if one if the modules fails we can repair only that one.
  2. 2. Kernel OS-The kernel OS can be further be classified into two categories,
    • 1. Monolithic Kernel- In this the operating system manages the system resources between the application and hardware but the user services and kernel services are implemented in the same address space.This approach makes the operating system very fast and due to this the size of the kernel increases but if one component stops working the whole system will crash.
    • 2. Micro-Kernel- In this the user services and kernel services are implemented in different address spaces. The user service is kept in user space and kernel service is kept in kernel space. This approach makes the operating system slow and due to this the size of the kernel is also less.
      On the other hand if one component stops working the whole system will not crash as the user service and kernel service is isolated. The interaction between the client program and the services running in user address space is is establised through message passing which makes it slow.
    • 3. Virtual Machines: A virtual machine (VM) is a software-based emulation of a physical computer that allows a single physical machine to operate as multiple virtual machines. There are different types of virtual machines, including:
      • 1. Type-1 VM or Hypervisor: This type of VM runs directly on the host's hardware to control the hardware and manage guest operating systems. Because it does not require a base operating system, it is often called a "bare-metal" hypervisor. Type-1 hypervisors are commonly used in enterprise environments for their efficiency and performance. Examples include VMware ESXi, Microsoft Hyper-V, and Xen.
      • 2. Type-2 VM or Hosted Hypervisor: This type of VM runs on a conventional operating system as an application. The host operating system manages the hardware, and the hypervisor provides virtualization services. Type-2 hypervisors are typically used in desktop and development environments where convenience and ease of use are prioritized over performance. Examples include VMware Workstation, Oracle VM VirtualBox, and Parallels Desktop.
